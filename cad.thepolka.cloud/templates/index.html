CAD Experience

I'm not a CAD expert. But I know CAD exists — and here's proof.

This is a small, honest project: a parametric phone/tablet stand, generated entirely from a Python script instead of clicked together in a feature tree. I didn't build it to claim CAD expertise — I built it because design systems thinking transfers to physical geometry, and I wanted something tangible to point at in that conversation.

The whole part is defined by a handful of named variables — lean angle, slot width, lip height — the same way a token file defines a component library. Change a number, rerun the script, every export regenerates in sync.

I'm also actively building out the rest of thepolka.cloud — an AI desktop overlay called Faire OS, a resume-generation tool, and open source contributions toward design-system tooling — while job-searching for Junior/Mid Developer and Design Systems Engineer roles.

Bottom line: I know CAD. I know it's not my primary lane — yet — but I can pick up a tool, understand its logic, and ship something real with it in a weekend. That's the pattern I bring to everything else here.
